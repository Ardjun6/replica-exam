# Guide - Plugins and Extensions

This guide explains extensions from the user’s point of view. It should make the model feel understandable: extensions add capabilities, permissions control access, and the user can disable anything that becomes risky or noisy.

## User concepts

- Install an extension, review permissions, enable it for a workspace, open its settings, and disable it when needed.
- Extension commands appear in command search with attribution.
- Custom views and panels should show which extension provides them.

## Safety habits

- Install reviewed extensions where possible.
- Read permission changes before updating.
- Remove extensions that ask for more access than the workflow needs.

## Review checklist

- The guide explains permissions in plain language.
- Users can find where extensions store settings.
- The guide links to marketplace, manifest, API, and security notes.
