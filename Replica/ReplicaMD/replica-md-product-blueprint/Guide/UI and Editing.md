# Guide - UI and Editing

This guide explains the day-to-day writing surface: opening pages, editing Markdown, using preview, navigating panes, and finding commands. It should feel like a practical tour of the workspace.

## Main workflow

- Open or create a page from command search.
- Write in Markdown with live preview where helpful.
- Use panes for reference material, backlinks, collections, or maps.

## Editing details

- Links, embeds, properties, code blocks, math, diagrams, and attachments all stay grounded in files.
- Search and quick open help users move without browsing every folder.
- Commands should be discoverable even before users learn hotkeys.

## Review checklist

- The guide links to editor and knowledge-layer notes.
- Examples use real Replica.md terms.
- The workflow can be followed without prior knowledge of the internals.
