# Guide - Styling and Theming

This guide shows how visual customization should work in Replica.md. It should be practical for users and still point contributors toward the CSS and theme contracts.

## User options

- Choose a theme, switch mode, enable snippets, and reset custom styling if something breaks.
- Use snippets for small adjustments and themes for complete visual systems.
- Keep accessibility settings above aesthetic preferences.

## Contributor notes

- Theme packages need manifests.
- CSS should use documented variables.
- Interface states such as focus, warnings, and unresolved links must stay visible.

## Review checklist

- Users can recover from a broken snippet.
- Theme docs link to app shell, editor, graph, and map CSS notes.
- Accessibility remains part of styling guidance.
