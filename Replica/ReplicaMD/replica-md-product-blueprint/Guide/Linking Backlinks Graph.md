# Guide - Linking Backlinks Graph

This guide explains how linking turns a folder of notes into a navigable workspace. It should be written for someone using Replica.md, not for someone reading system internals.

## User flow

- Create a link while writing.
- Open the backlink panel to see where the current page is referenced.
- Use the relationship graph to inspect clusters, orphans, and nearby pages.

## Good habits

- Use clear page titles.
- Link when the relationship helps future navigation.
- Review unresolved links during cleanup instead of treating them as mistakes.

## Review checklist

- A user can follow the guide without knowing the data model.
- The guide links to the backlink and graph implementation notes.
- Examples use Replica.md terminology consistently.
