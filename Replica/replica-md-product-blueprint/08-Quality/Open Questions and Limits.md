# Open Questions and Limits

This page keeps uncertainty visible. It is better to name limits early than to imply Replica.md already has answers for every platform, sync edge case, extension risk, and publishing workflow.

## Questions to settle

- How far mobile editing should go in the first release.
- Which extension capabilities should be postponed until the permission model is proven.
- How much query language complexity users actually need for collections.

## Known limits

- Very large workspaces may require staged indexing and partial graph rendering.
- Hosted sync needs more security review before public availability.
- Marketplace review can start small before opening to broad submissions.

## Review checklist

- Unresolved questions have owners or next steps.
- Limits are not hidden in feature pages.
- Decisions move from this page into the decision log when settled.
