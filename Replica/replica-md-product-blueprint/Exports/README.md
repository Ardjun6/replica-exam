# Exports

Exports are the way users take work out of Replica.md in a deliberate format. The export folder should be easy to understand and safe to delete or regenerate.

## Export types

- Static site output, PDF or print-ready pages, Markdown bundles, and selected attachment packages.
- Export metadata that records source pages, date, settings, and warnings.
- Clear handling for unresolved links and excluded attachments.

## Folder rules

- Generated output should not be mistaken for source notes.
- Exports should be reproducible from workspace files and settings.
- Private pages stay out unless explicitly selected.

## Review checklist

- Exports can be opened without Replica.md where the format allows.
- The export report lists skipped or broken items.
- Generated files do not overwrite user notes.
