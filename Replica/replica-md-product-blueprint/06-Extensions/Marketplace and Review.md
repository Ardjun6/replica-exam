# Marketplace and Review

The marketplace should make extensions discoverable without turning trust into guesswork. Review is part product quality, part security boundary, and part user education.

## Listing content

- Name, author, description, version, permissions, screenshots, compatibility, changelog, and support links.
- Clear badges for reviewed, experimental, desktop-only, or permission changes.
- Install and update flows that surface important access changes.

## Review checks

- Permission scope, code behavior, dependency licenses, network access, data storage, and uninstall cleanup.
- Manual review for extensions that touch files, sync, publishing, or external services.
- Fast removal path for harmful or abandoned listings.

## Review checklist

- Users can compare extensions without opening source code.
- Permission changes require renewed consent.
- Review records are tied to extension versions.
