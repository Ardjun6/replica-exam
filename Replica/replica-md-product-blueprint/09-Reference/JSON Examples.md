# JSON Examples

JSON examples document the shapes Replica.md writes into profile files, maps, collections, themes, and extensions. They should be small enough to read and complete enough to validate.

## Example coverage

- Workspace profile settings, graph options, hotkeys, sync settings, collection files, map files, extension manifests, and theme manifests.
- Use realistic field names from the product.
- Include schema versions where the real files require them.

## Quality bar

- Keep formatting consistent.
- Avoid optional fields unless the example explains them.
- Link examples to the feature notes that own the format.

## Review checklist

- Examples match files shipped in the package.
- Invalid examples are clearly labeled as invalid.
- Schema changes update examples in the same pass.
