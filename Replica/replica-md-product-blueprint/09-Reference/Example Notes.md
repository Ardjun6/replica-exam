# Example Notes

Example notes show what ordinary user content might look like inside a Replica.md workspace. They should be simple, believable, and useful for testing backlinks, tags, properties, and search.

## Example set

- A short concept note.
- A technical learning note.
- A daily note with tasks and links.
- A small inbox note that can be processed later.

## Testing value

- Use example notes to confirm link parsing, backlinks, properties, templates, and search results.
- Keep them free of private information.
- Prefer realistic wording over lorem ipsum.

## Review checklist

- Examples create visible backlinks.
- Frontmatter and links are valid.
- The notes still make sense outside the app.
