# Example Map

The example map gives contributors a quick visual model of how Replica.md can connect strategy, architecture, core systems, and quality work. It should stay small enough to understand at a glance.

## Map contents

- Strategy node connected to architecture and core system notes.
- Quality node connected to security, performance, and test strategy.
- Release node connected to sync, publishing, and packaging.

## Use

- Open the map to check whether the Map UI and Map JSON behavior agree.
- Use it as a fixture for canvas rendering and link resolution.
- Update it when major documentation areas change.

## Review checklist

- Every node resolves to a real page or documented placeholder.
- Edges use clear relationship labels.
- The map remains useful as a smoke test.
