# Code Snippets

Code snippets in the reference section should be short, accurate examples that help contributors understand the shape of Replica.md contracts. They are not production code by default.

## Example rules

- Show one idea per snippet.
- Name placeholders clearly and avoid secrets, tokens, or machine-specific paths.
- Link each snippet to the feature page it supports.

## Useful examples

- IPC request and response shapes.
- Extension command registration.
- Collection query samples.
- Renderer or parser edge cases.

## Review checklist

- Examples compile or are clearly marked as pseudocode.
- Snippet language labels are correct.
- Examples do not contradict schema files.
