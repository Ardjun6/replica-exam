# Product Blueprint

Replica.md is being shaped as a plain-file workspace for people who write, connect, review, and ship knowledge from the same folder. The product is not defined by comparison. It is defined by a simple promise: the user owns the files, the app keeps the relationships readable, and every advanced view can be traced back to Markdown on disk.

## Product line

- Local workspaces stay useful even without an account or network connection.
- Pages, attachments, collections, maps, and exports belong to the workspace rather than to a hosted database.
- Backlinks and graph views are treated as navigation surfaces, not decorative extras.

## Build order

- Start with file safety, indexing, editing, and search before adding richer views.
- Use the same terms in the interface, schema files, documentation, and extension examples.
- Keep the first release narrow enough that writing and linking feel dependable.

## Review checklist

- A new contributor can describe the product without mentioning competing apps.
- Each major feature has a clear owner, contract, and related implementation note.
- Product language stays consistent across README, guides, schemas, and examples.
