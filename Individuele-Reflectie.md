# Individuele reflectie
## Wat is mijn leerdoel?
Mijn leerdoel was om beter te leren plannen, vooral voor school, maar ook in het algemeen. Ik heb voor dit leerdoel gekozen omdat ik eerder vaak moeite had om deadlines op tijd te halen zonder stress te ervaren. Vroeger kwam ik regelmatig in tijdsnood, waardoor ik taken soms te laat of minder goed afrondde. 
Sinds ik mijn planning beter op orde heb en duidelijk overzicht heb van wat ik moet doen, merk ik dat schoolwerk makkelijker en rustiger verloopt.

## Hoe heb ik het aangepakt?
In het begin ben ik online gaan zoeken naar manieren om mijn planning beter bij te houden. Ik dacht eerst aan Google Calendar, omdat veel bedrijven deze tool ook gebruiken. Toch liep ik al snel ergens tegenaan. Ik probeerde de iCal van school toe te voegen, maar dat vond ik onduidelijk en niet erg gebruiksvriendelijk van de google calendar. Daarna heb ik ook Apple Calendar geprobeerd, maar die vond ik ook niet prettig werken.

Toen kwam ik op het idee om zelf een applicatie te maken die beter past bij mijn eigen manier van werken. Uiteindelijk heb ik een eigen applicatie gemaakt waarin ik ook notities kan bijhouden voor verschillende vakken en prive dingen lokaal.je kan deze notities en calendar ook commiten naar github en als je uiteindelijk naar een andere device gaat kan je het weer clonen vanuit de applicatie zelf. Ook kan ik mijn planning beheren met behulp van een markdown-bestand en JSON. Hierdoor heb ik meer controle over mijn planning en kan ik het systeem aanpassen aan wat voor mij het beste werkt.

## Bewijslast

Als bewijslast kan ik laten zien dat ik actief aan mijn leerdoel heb gewerkt door mijn eigen planningstool te bouwen en te gebruiken. Hierbij horen bijvoorbeeld:

### Bewijs 1: Screenshot van de applicatie
Hier kan ik laten zien hoe mijn planning eruitziet binnen de applicatie. Dit bewijst dat ik niet alleen heb nagedacht over plannen, maar ook een systeem heb gemaakt dat ik echt kan gebruiken.

### Bewijs 2: Screenshot van mijn notities per vak
Hiermee kan ik aantonen dat ik mijn schoolwerk beter probeer te organiseren. Door notities per vak bij te houden, heb ik sneller overzicht en kan ik makkelijker terugvinden wat ik moet doen.

### Bewijs 3: Screenshot van GitHub commits
De commits laten zien dat ik mijn planning en notities opsla en bijhoud. Dit laat ook zien dat ik bewust bezig ben geweest met structuur, versiebeheer en het veilig bewaren van mijn werk.

### Bewijs 4: Voorbeeld van markdown of JSON-planning
Een voorbeeld van mijn planning in markdown of JSON laat zien hoe ik mijn taken overzichtelijk maak. Dit bewijst dat ik een manier heb gevonden die goed aansluit bij mijn eigen werkwijze.

## Peer-feedback

Ik heb ook peer-feedback meegenomen in mijn reflectie. Uit feedback van anderen kwam naar voren dat mijn idee om een eigen applicatie te maken goed past bij mijn leerdoel, omdat ik hiermee een persoonlijke oplossing heb gemaakt voor een probleem waar ik echt tegenaan liep. In plaats van alleen een bestaande tool te gebruiken, heb ik onderzocht wat wel en niet voor mij werkte en daarna zelf iets gebouwd.

Een punt uit de feedback was dat ik ervoor moet zorgen dat mijn planning niet te ingewikkeld wordt. Als ik te veel functies toevoeg, kan het systeem juist onoverzichtelijk worden. Deze feedback vond ik nuttig, omdat mijn doel juist is om meer rust en overzicht te krijgen. Daarom probeer ik de applicatie simpel en praktisch te houden, met vooral functies die ik echt nodig heb.

Ook kreeg ik terug dat het goed is dat ik mijn planning combineer met notities. Daardoor heb ik niet alleen deadlines in beeld, maar ook de inhoudelijke informatie die bij vakken en opdrachten hoort. Dit helpt mij om sneller te schakelen tussen plannen en uitvoeren.

## Hoe sta ik er nu voor?

Op dit moment sta ik er beter voor dan aan het begin van het semester. Ik merk dat ik bewuster nadenk over mijn planning en minder snel alles op het laatste moment doe. Doordat ik mijn taken overzichtelijker heb staan, weet ik beter wat prioriteit heeft. Ook helpt het dat ik mijn planning zelf kan aanpassen aan mijn eigen manier van werken.

Ik ben nog niet perfect in plannen. Soms schuif ik taken nog steeds door of onderschat ik hoeveel tijd iets kost. Wel merk ik dat ik dit sneller doorheb dan eersst. Door mijn planning minimaal 1x per dag te bekijken.

Mijn grootste vooruitgang is dat ik niet meer alleen probeer te onthouden wat ik moet doen. Ik heb nu een systeem waarin ik taken, deadlines en notities kan vastleggen. Daardoor zit er minder druk in mijn hoofd en kan ik rustiger werken. later zal ik nog een (local) ai toevoegen waarmee ik kan bijvoorbeeld vragen over bijvoorbeeld `wat zijn de taken voor deze week en om hoelaat zijn ze` en dan geeft het mij een mooie tabel met alles over deze week.

## Wat heb ik geleerd? en wat moet ik nog doen om de applicatie te verbeteren?

Ik heb geleerd dat plannen niet alleen betekent dat je deadlines opschrijft. Een goede planning moet ook passen bij hoe je zelf werkt. Voor mij werkte Google Calendar en Apple Calendar niet goed genoeg, omdat ik daar niet genoeg controle en overzicht in ervaarde. Door zelf een applicatie te maken, kon ik een systeem bouwen dat beter aansluit bij mijn eigen behoeften, waarbij ik de vorm van obsidian heb overgenomen en een paar functies van notion. Later is de bedoeling dat ik wat meer in de richting van plugins gaat en kan ik dit online zetten waarbij anderen kan ideeen kan geven.
